# 需求分析文档

## 1. 项目定位

本系统面向建筑能源管理与运维场景，目标是将数据管理、查询统计、可视化分析、MCP 数据接入和智能问答整合到统一系统中，形成课程项目的完整原型。Web 平台面向人工演示和运维操作，MCP Server 面向支持 MCP 的 AI 客户端和智能体调用。

## 2. 目标用户

- 项目评审老师：关注功能闭环、系统完整性和展示效果。
- 建筑运维人员：关注能耗查询、异常识别和运维辅助。
- AI 客户端或智能体：通过 MCP 协议调用数据查询、统计分析、异常诊断和知识库问答能力。
- 项目团队成员：关注数据结构、接口规范和协作效率。

## 3. 核心业务问题

- 多源能耗数据分散，缺少统一口径。
- 人工统计效率低，跨建筑和跨时段分析困难。
- 异常原因判断依赖经验，缺少辅助说明。
- 传统系统交互方式不自然，演示说服力不足。

## 4. 功能需求

### 4.1 数据层

- 支持导入结构化建筑能耗数据。
- 提供字段校验和统一数据字典。
- 预留原始数据与清洗后数据存储目录。

### 4.2 查询与统计

- 支持按建筑、时间区间检索记录。
- 支持通过 MCP Tools 按建筑、楼层、时间区间和条数查询记录。
- 支持时段汇总分析。
- 支持 COP 计算分析。
- 支持异常识别与异常明细查看。

### 4.3 可视化

- 提供趋势类图表。
- 提供建筑间对比图表。
- 提供演示首页与关键指标卡片。

### 4.4 智能运维

- 建立知识库目录和初始文档。
- 提供问答接口。
- 支持知识检索、本地规则回答和可选外部大模型响应。
- 支持 MCP 客户端调用智能问答、异常解释和运营报告工具。

## 5. 非功能需求

- 系统能稳定演示核心流程。
- 页面结构清晰，适合课堂展示。
- 代码结构可扩展，便于后续补功能。
- REST API 与 MCP Server 均可被测试和复现。
- 文档齐备，便于答辩和提交。

## 6. MVP 范围

当前仓库优先保证以下闭环：

1. 样例数据可读可查。
2. 后端核心接口可访问。
3. 前端首页可展示项目骨架。
4. MCP Server 可暴露数据接入、查询统计、异常诊断和智能问答工具。
5. 文档和知识库结构已准备。

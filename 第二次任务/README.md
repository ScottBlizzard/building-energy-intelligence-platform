# 第二次任务包

这个文件夹是给第二次任务分发专门准备的。

第二轮和第一轮一样，仍然坚持一个原则：

**三个人各做各的一块，互相不要交叉写文件，最后由许奕统一做第二次整合。**

## 这个文件夹里有什么

- [`00-统一说明.md`](00-统一说明.md)：第二轮任务的共同背景、目标和边界
- [`01-给王天一-后端与架构.md`](01-给王天一-后端与架构.md)：王天一的第二次任务
- [`02-给马源胜-数据与AI.md`](02-给马源胜-数据与AI.md)：马源胜的第二次任务
- [`03-给周由-前端与集成.md`](03-给周由-前端与集成.md)：周由的第二次任务
- [`04-统一提交要求.md`](04-统一提交要求.md)：三个人统一遵守的提交和说明要求
- [`05-给许奕-整合说明.md`](05-给许奕-整合说明.md)：第二次整合时参考的说明

## 你怎么用

1. 先把 [`00-统一说明.md`](00-统一说明.md) 和 [`04-统一提交要求.md`](04-统一提交要求.md) 发给三个人。
2. 再分别把对应人的任务文件单独发给他。
3. 明确告诉他们：第二轮不是重新发明系统，而是在第一次整合后的统一底座上，继续补强自己负责的模块。
4. 第二轮结束后，你按 [`05-给许奕-整合说明.md`](05-给许奕-整合说明.md) 去收成果。

## 第二轮任务的核心原则

- 不允许为了做新功能去改坏第一次整合后的稳定能力。
- 每个人只改自己负责的目录，不交叉写别人的区域。
- 前端第二轮默认只消费**当前已经存在的稳定接口**，不要等后端新增接口才能完成。
- 后端第二轮可以新增能力，但不要擅自改旧接口字段和路径。
- 知识与数据第二轮继续只交素材和结构化资料，不直接改主样例数据和数据字典。

# 27 · 答辩演示指导脚本（按真实界面）

> 这份脚本严格对照系统**真实的页签名字和按钮文字**，照着点就行。
> 关键提示：**几乎所有"决策"相关的内容都在「工单中心」这一个页签里**，需要往下滚动；系统里**没有**单独的"经济决策"页签。

---

## 一、先认识界面（界面地图）

登录后顶部有一条 **「运营沙盘 · 时间机器」** 控制栏（管理员可见），下面是页签区。

### 顶部时间机器控制栏（管理员）
按钮（从左到右）：
- **启动沙盘（定位到 5 月）** —— 把"今天"定位到 2026-05-01。
- **▶ 下一天** / **⏭ 下一周** —— 推进模拟时间。
- **重置时钟** —— 只把时间机器复位（不动工单/预算）。
- **⟲ 重置演示（回到初始）**（右侧红色按钮）—— 一键清空所有工单+预算、复位时间机器、并预置 3 条历史样例工单。**每次开演前先点它**。

### 管理员页签（admin 登录后）
| 页签 | 里面有什么 | 演示要点 |
|---|---|---|
| **总览** | KPI 卡片、"管理员业务闭环看板"（下一步动作 / 待复核工单）、楼宇风险三维态势 | 开场展示系统全貌 |
| **数据浏览** | 原始能耗数据表 + 导出 CSV | 一句话带过即可 |
| **统计分析** | 异常分析工作台：按建筑/楼层/时间锁定异常、异常解释、趋势、设备监测、全局对比/COP 排名 | 展示"能发现问题" |
| **工单中心** ⭐ | 建单派单 + **决策优先级/派单建议** + **异常处置三情景对照** + 工单全生命周期 | **全场核心，内容多，要往下滚** |
| **预算管理** | 预算生成与执行分析（BudgetPanel） | 财务视角 |
| **改造分析** | ROI 改造测算与方案对比（ROIPanel） | 投资回报视角 |
| **决策报告** | 未来 7 天能耗预测 + 节能策略模拟器 + **自动运营报告** | 适合收尾汇报 |
| **智能问答** | 接地到真实业务的 LLM 助手 | 展示"可信助手" |

## 二、演示流程（按顺序照做）

### 第 0 步 · 开场准备（10 秒）
登录 `admin / admin123` → 点顶部红色 **「⟲ 重置演示（回到初始）」** → 等待提示"演示已回到初始状态…"。
> 现在系统是干净初始态，且已有 3 条历史样例工单（供"相似案例/报告"用）。

### 第 1 步 · 总览 + 启动时间机器（约 2 分钟）
1. 停在 **总览** 页签，介绍 KPI 和"业务闭环看板"。
2. 点顶部 **「启动沙盘（定位到 5 月）」**，时钟显示 `2026-05-01`。
3. 点一两次 **「▶ 下一天」**，让老师看到日期推进、数据边界变化、可能冒出新异常。
> 话术："指针之前是已发生历史，之后是'未来'。我现在的处置会改变系统的未来——这是我们和普通监控大屏最大的不同。"

### 第 2 步 · 统计分析：发现问题（约 1.5 分钟）
进 **统计分析** 页签：选建筑/楼层 → 看"异常明细"和"异常解释面板"。
> 话术："系统能基于同建筑基线识别电耗异常、COP 偏低、夜间高负荷等。"

### 第 3 步 · 工单中心：决策 + 三情景对照（约 2.5 分钟）⭐
进 **工单中心** 页签：
1. 顶部是建单区：**选择建筑 → 选择楼层 → 选择异常**（下拉）。
2. **选中一条异常后**：
   - 上方出现该异常的风险分/估算浪费/可回收/SLA 预览；
   - **往下滚动**，会看到 **「异常处置三情景对照」**：不处理 / 立即处理 / 延迟 3 天处理 在未来 7 天的曲线差异和决策结论句。
   > 话术："系统用反事实模拟量化了'拖延的代价'。"
3. 再往下是 **「今日资源约束派单建议」** 和决策优先级列表（按风险/损失/SLA/碳排/重复异常综合打分）。
   > 话术："就算现在还没建工单，这里也会把**当前实时异常**当候选排好序，告诉我先修哪台最划算。"

### 第 4 步 · 工单闭环 + 因果干预（约 4 分钟，重头戏）
仍在 **工单中心**：

**4.1 管理员建单/派单**
- 选中一条异常 + 在"分配工人"选好人 → 点 **「生成并派单」**；
- 或点蓝色 **「🔄 一键生成待确认工单队列」** 把所有异常批量转成待确认工单。
- 往下滚到工单列表，对"待确认"工单点 **「确认派单」**（或"忽略告警"）。
- 说明状态机：待确认 → 已派单 → 处理中 → 待复核 → 已关闭。

**4.2 切换到工人处理**
- 右上角 **「退出登录」** → 登 `worker_ahu / worker123`。
- 进 **我的工单**：对工单点 **「接单并开始处理」** → 填写"现场实际原因/处理结果"等 → 点 **「提交管理员复核」**。
  > 指出右侧"历史相似案例"有内容（因为开场预置了样例工单）。
  > 指出"修复后能耗/COP"标注的是 **预计值（估算）**，不冒充实测。

**4.3 切回管理员复核 + 看因果效果**
- 退出登录 → 登 `admin / admin123` → 进 **工单中心** 找到"待复核"工单 → 填复核意见 → 点 **「复核通过并关闭」**（或"驳回重办"）。
- 回到顶部点 **「▶ 下一天 / ⏭ 下一周」**。
  > 话术："注意这台刚关闭的设备，从处置日往后**不再异常**——因为关闭工单触发了对它未来数据的因果干预；而**没处理的设备异常会继续**。这就是'我的操作改变系统未来'。"

### 第 5 步 · 决策报告（约 1 分钟）
进 **决策报告** 页签 → 点 **「生成报告」** 展示自动运营报告（含节能收益，估算项有"预计"标注）；可顺带玩一下"节能策略模拟器"。

### 第 6 步 · 预算 / 改造分析（可选，约 1 分钟）
进 **预算管理** / **改造分析**，展示预算执行分析与 ROI 测算。

### 第 7 步 · 智能问答（约 1.5 分钟）
进 **智能问答**，问："现在有哪些高优先级工单？建议先处理哪台设备？"
> 话术："助手回答**接地到真实工单/异常数据**，并校验引用的实体 ID，避免编造。"

### 第 8 步 · 权限隔离佐证（可选，约 1 分钟）
登 `worker_ahu`，让老师看到工人**只有"我的工单/智能问答"两个页签**，没有决策/预算/ROI。
> （后端层面：工人操作他人工单/自我复核会被拒，返回 403/409。）

### 收尾
一句话总结 + 如需重演：点 **「⟲ 重置演示（回到初始）」** 即可秒回初始。

## 四、常见疑问（自检）

- **"找不到决策面板？"** → 它不是独立页签，在 **工单中心** 里往下滚。
- **"三情景对照不显示？"** → 必须先在工单中心"选择异常"下拉里**选中一条异常**才会出现。
- **"相似案例为空？"** → 确认开场点过"重置演示"（它会预置历史样例工单）。
- **"修了的设备还在异常？"** → 因果干预从**处置当天**起生效，记得点"复核通过并关闭"后再"下一天"。

## 附：本轮为答辩做的改动

- **性能优化**：对"可见数据集 + 运营标注帧"按仿真指纹做了缓存。实测一次"下一天"的接口扇出从约 10–20 秒降到约 1.4 秒（首个请求）/ 0.4 秒（同日再次），预算/改造分析从约 10 秒降到毫秒级。
- **演示一键重置**：顶部红色按钮 + 后端 `POST /api/v1/demo/reset`。
- **相似历史案例种子**：预置 AHU/FCU/CH 三类历史已关闭工单。
- **决策面板空态回退**：无在处理工单时用当前可见异常生成候选优先级。
- **管理端权限一致化**：前端登录后自动携带 `operator_id`，后端统一校验。

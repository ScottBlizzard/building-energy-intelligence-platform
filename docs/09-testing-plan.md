# 测试计划

## 1. 测试目标

确保系统在课程演示场景下具备基本可用性和稳定性，优先覆盖查询、分析、问答和页面联调主流程。

## 2. 测试范围

- 后端接口可访问性
- 数据加载与字段完整性
- 主要统计逻辑结果格式
- 前端页面基础渲染
- 问答接口响应结构

## 3. 测试层次

### 3.1 后端接口测试

已在 `backend/tests/` 中预留：

- 健康检查测试
- 数据接口测试
- 分析接口测试
- 问答接口测试

### 3.2 人工联调测试

- 打开首页是否能看到总览
- 是否能筛选建筑并看到记录变化
- 是否能看到 COP 排名和异常列表
- 问答输入后是否返回回答与引用

### 3.3 演示彩排测试

- 后端启动后接口是否稳定
- 前端页面是否无明显报错
- 演示脚本是否与页面一致
- 是否存在空白页面、死链接或错误文案

## 4. 当前已知限制

- 本地环境未安装 `pytest`，所以测试文件已补齐，但尚未在当前机器执行完整测试。
- 前端尚未接入正式图表库，当前以结构化页面和轻量可视占位为主。

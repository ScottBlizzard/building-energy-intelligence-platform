## 你这轮的独立任务

### 任务 A：把知识素材做成“更适合检索”的结构

第一轮知识库已经能引用，但还不够像后续 RAG 能直接吃进去的素材。

第二轮建议你补一套**检索化素材**，核心是：

- 让问题和知识片段的对应关系更清楚
- 让后端以后更容易做检索匹配
- 让演示问题和知识来源之间形成稳定映射

建议新增文件：

- `knowledge_base/faq/rag_retrieval_cards_round2.md`
- `knowledge_base/faq/assistant_eval_round2.md`

推荐内容结构：

- 问题
- 关键词
- 应命中的知识文件
- 应命中的核心段落主题
- 预期回答要点
- 适不适合课堂演示

### 任务 B：把数据增强方向做成结构化“候选资料包”

第二轮你仍然不要改主样例 CSV，但你应该把下一轮最值得增强的数据方向整理成结构化资料。

建议你重点补：

- 节假日 / 人员密度 / 建筑面积 / 多设备 / 连续异常这些方向

建议新增文件：

- `data/processed/round2_dataset_upgrade_plan.md`
- `data/processed/anomaly_scenario_catalog_round2.csv`
- `data/processed/building_profile_reference_round2.csv`

这些文件最好做到：

- 不是空泛建议
- 而是“字段 / 场景 / 建议值 / 参考依据 / 为什么值得加”

### 任务 C：补一套问答评测素材

第一轮是“能问什么”；第二轮建议你补“问得对不对、引得准不准”的评测素材。

建议新增文件：

- `data/processed/qa_evaluation_set_round2.csv`

建议字段：

- `question_id`
- `question`
- `question_type`
- `expected_sources`
- `expected_keywords`
- `expected_answer_points`
- `demo_priority`

这份东西后面不管是后端做检索，还是许奕自己验收问答质量，都会非常有用。

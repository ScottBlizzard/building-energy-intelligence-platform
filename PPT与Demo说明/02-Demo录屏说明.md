# Demo 录屏说明

本文档给负责系统 Demo 和录屏的同学使用。目标是录出一段能证明系统完整可运行、核心功能可交互、文档材料可对应的演示视频。建议时长 6 到 8 分钟，如果课程展示时间更长，可以扩展到 10 分钟左右。

## 1. 录屏前准备

### 1.1 拉取最新项目

在项目目录执行：

```powershell
git pull origin main
```

项目目录一般是：

```text
D:\软件经济管理
```

### 1.2 准备环境变量

如果需要演示外部大模型问答，请向项目负责人获取真实 `.env` 文件，并放在项目根目录：

```text
D:\软件经济管理\.env
```

注意：

1. 不要把 `.env` 文件上传到 GitHub。
2. 不要在录屏中打开 `.env`。
3. 不要在视频里展示 API Key。
4. 如果没有真实 `.env`，可以复制 `.env.example` 为 `.env`，系统主体功能仍然可以演示。

### 1.3 启动后端

打开第一个 PowerShell 窗口，在项目根目录执行：

```powershell
.\scripts\start-backend.ps1
```

如果脚本无法运行，可以使用手动方式：

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r backend\requirements.txt
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000 --app-dir backend
```

后端启动后检查：

```text
http://127.0.0.1:8000/api/v1/health
```

如果返回健康状态，说明后端正常。

### 1.4 启动前端

打开第二个 PowerShell 窗口，在项目根目录执行：

```powershell
.\scripts\start-frontend.ps1
```

如果脚本无法运行，可以使用手动方式：

```powershell
cd frontend
npm install
npm run dev
```

前端地址：

```text
http://127.0.0.1:5173
```

### 1.5 可选：启动 MCP 服务

如果需要展示 MCP 或扩展工具能力，可以打开第三个 PowerShell 窗口执行：

```powershell
.\scripts\start-mcp.ps1
```

普通 Demo 不强制展示 MCP，优先保证主系统流畅。

## 2. 录屏软件建议

可以使用以下任意一种方式：

1. OBS：画质最好，适合录完整浏览器窗口。
2. Windows Xbox Game Bar：按 `Win + Alt + R` 开始或停止录制。
3. PowerPoint 自带屏幕录制：适合和 PPT 合并使用。

录屏设置建议：

1. 分辨率使用 1920 x 1080。
2. 浏览器缩放设置为 90% 或 100%。
3. 录制浏览器窗口或完整屏幕都可以，但不要露出私人信息。
4. 录制前关闭无关软件、聊天窗口和浏览器标签。
5. 鼠标移动要慢，点击前可以停顿半秒。
6. 如果讲错或点错，不要硬接，可以停下重新录这一段。

## 3. 推荐 Demo 路线

下面是一条完整、稳定、适合录屏的演示路线。建议录屏时严格按这个顺序操作。

### 第 1 段：系统启动与项目定位

操作：

1. 打开浏览器访问 `http://127.0.0.1:5173`。
2. 展示系统首页。
3. 可以快速切到 `http://127.0.0.1:8000/api/v1/health`，证明后端正常。

讲解词参考：

```text
这是我们的建筑能耗智能分析与异常工单平台。系统面向校园建筑能耗管理场景，目标是把能耗数据查询、异常识别、楼层定位、工单处理和管理报告连接成一个闭环。当前前端和后端都在本地启动，后端健康接口正常返回，说明系统已经可以完整运行。
```

### 第 2 段：数据总览与 3D 楼宇

操作：

1. 回到系统首页。
2. 展示关键指标卡片。
3. 展示 3D 楼宇视图。
4. 拖动旋转 3D 楼宇。
5. 指出红色楼层和绿色楼层。

讲解词参考：

```text
首页展示了系统的总体运行状态，包括记录数量、异常情况和建筑健康状态。中间的三维楼宇视图可以旋转查看，不同楼层用颜色表示风险状态。绿色表示当前楼层健康，红色表示该楼层存在需要关注的异常。
```

### 第 3 段：点击异常楼层联动统计分析

操作：

1. 在 3D 楼宇中点击一个红色异常楼层。
2. 等待页面跳转到统计分析。
3. 展示筛选条件已自动锁定到对应建筑和楼层。
4. 展示异常明细列表。

讲解词参考：

```text
点击异常楼层后，系统会直接跳转到统计分析页面，并把分析范围锁定到对应建筑和楼层。这样管理人员不需要在大量数据里人工查找，可以直接看到该楼层的异常记录和分析结果。
```

### 第 4 段：异常明细与异常解释

操作：

1. 在统计分析页面查看“选定范围异常明细”。
2. 选择一条异常记录。
3. 展示异常原因、指标证据、严重程度和处理建议。
4. 如果页面有趋势或原因统计图，也可以顺带展示。

讲解词参考：

```text
统计分析页面优先展示选定范围内的异常明细。每条异常不只是显示状态，还会给出触发规则、关键指标证据和处理建议。例如空调电耗异常、COP 偏低或者夜间负荷异常，都可以被系统识别出来，并转化成运维人员能理解的解释。
```

### 第 5 段：健康楼层展示逻辑

操作：

1. 回到 3D 楼宇或统计分析筛选区。
2. 选择一个绿色健康楼层。
3. 展示页面只显示健康状态，不展示异常模块。

讲解词参考：

```text
如果选择的是健康楼层，系统不会强行展示空的异常图表，而是直接提示该楼层状态健康。这样可以避免页面出现大量无意义的暂无数据，让分析页面更符合实际业务逻辑。
```

### 第 6 段：数据浏览与查询

操作：

1. 切换到数据浏览页面。
2. 按建筑、楼层或时间范围筛选。
3. 展示表格字段。
4. 如果有导出按钮，可以点击展示 CSV 导出功能。

讲解词参考：

```text
数据浏览页面主要用于查看原始记录，它和统计分析的定位逻辑分开。这里可以按建筑、楼层和时间范围查询全量数据，也可以导出 CSV，便于后续报告整理或进一步分析。
```

### 第 7 段：创建异常工单

操作：

1. 切换到异常工单处理中心。
2. 在创建区域选择建筑。
3. 选择楼层。
4. 选择一条异常。
5. 选择负责人。
6. 点击创建工单。
7. 展示新工单进入处理中状态。

讲解词参考：

```text
工单中心负责把异常分析结果转化成后续处理任务。这里可以选择某个建筑、某个楼层下的具体异常，再分配给对应负责人。创建后工单直接进入处理中状态，并以黄色显示，同时排到列表前面，方便优先处理。
```

### 第 8 段：完成工单并验证持久化

操作：

1. 点击刚创建工单的完成按钮。
2. 展示工单变成绿色已完成状态。
3. 刷新浏览器页面。
4. 回到工单中心，展示状态仍然保留。

讲解词参考：

```text
当运维人员处理完成后，可以直接点击完成。完成后的工单变为绿色，并移动到列表后方。这里刷新页面后状态仍然保留，说明工单不是临时前端效果，而是已经写入运行数据，实现了基本的处理闭环。
```

### 第 9 段：决策报告

操作：

1. 切换到决策报告页面。
2. 展示风险摘要、重点建筑、优化建议和管理结论。

讲解词参考：

```text
决策报告面向管理人员，不要求他们逐条查看原始数据，而是把异常情况、风险建筑和优化建议汇总起来。这样系统不仅能发现问题，还能支持管理决策和后续节能改造。
```

### 第 10 段：AI 助手

操作：

1. 打开 AI 助手区域。
2. 如果配置了真实 `.env`，选择一个可用模型。
3. 输入一个准备好的问题。
4. 展示模型回答。

推荐问题：

```text
请分析综合教学楼A屋顶楼层最近的能耗异常风险，并给出运维处理建议。
```

讲解词参考：

```text
系统还接入了 AI 辅助解释能力。管理人员可以用自然语言提问，系统会结合能耗分析结果给出更容易理解的说明和建议。真实模型密钥通过本地环境变量配置，不会进入代码仓库。
```

如果没有外部模型可用，可以这样讲：

```text
当前演示环境没有展示真实 API Key，因此 AI 部分可以使用系统内置分析结果完成说明。项目代码已经保留外部模型配置接口，部署时只需要在本地 `.env` 中配置密钥即可。
```

### 第 11 段：正式材料与测试证据

操作：

1. 打开项目文件夹中的 `期末提交材料/documents/pdf-latex`。
2. 展示 SRS、SDD、SEE、SEM、测试报告等 PDF。
3. 可以快速展示 `README.md` 中的运行说明。

讲解词参考：

```text
除了系统本身，项目也整理了正式提交材料，包括需求规格说明书、软件设计说明书、软件经济分析、软件工程管理说明、测试与验收报告和用户手册。系统、文档和测试材料是一一对应的。
```

### 第 12 段：收尾总结

操作：

1. 回到系统首页或 3D 楼宇页面。
2. 停留 3 到 5 秒。

讲解词参考：

```text
整体来看，本项目已经完成从能耗数据读取、统计分析、异常定位、三维楼层展示、工单处理到管理报告的完整流程。它既能作为课程项目展示，也具备进一步扩展到真实建筑能耗管理场景的基础。
```

## 4. 录屏时不要做的事

1. 不要打开 `.env` 文件。
2. 不要展示 API Key。
3. 不要打开私人聊天、邮箱、网盘或无关项目。
4. 不要在视频中长时间等待安装依赖。
5. 不要现场调试报错，录屏前先确认系统已经正常。
6. 不要把鼠标快速乱晃，影响观看。
7. 不要录太长，核心功能讲清楚即可。

## 5. 常见问题处理

### 5.1 端口 8000 被占用

现象：后端启动失败，提示端口已被占用。

处理方式：

1. 关闭之前的后端 PowerShell 窗口。
2. 或在任务管理器中结束旧的 Python 进程。
3. 再重新运行 `.\scripts\start-backend.ps1`。

### 5.2 端口 5173 被占用

现象：前端提示端口被占用。

处理方式：

1. Vite 可能会自动换到 5174。
2. 以命令行输出的地址为准。
3. 浏览器打开实际显示的本地地址。

### 5.3 前端页面空白

处理方式：

1. 确认后端已经启动。
2. 刷新浏览器。
3. 回到 `frontend` 目录重新执行 `npm install` 和 `npm run dev`。
4. 如果仍有问题，先运行 `.\scripts\check-project.ps1` 看错误位置。

### 5.4 AI 助手没有回答

处理方式：

1. 检查根目录是否有 `.env`。
2. 确认 `.env` 中配置了可用模型密钥。
3. 不要在录屏中展示密钥内容。
4. 如果外部模型暂时不可用，跳过 AI 真实调用，说明系统主体功能不依赖外部模型。

### 5.5 工单刷新后没有保留

处理方式：

1. 确认后端正在运行，而不是只打开了前端。
2. 创建工单后等待页面提示完成。
3. 再刷新页面验证。
4. 如果仍然异常，重新启动后端再录制工单部分。

录屏前确认：

1. 后端能打开 `http://127.0.0.1:8000/api/v1/health`。
2. 前端能打开 `http://127.0.0.1:5173`。
3. 3D 楼宇可以旋转和点击。
4. 统计分析能显示异常明细。
5. 健康楼层只显示健康状态。
6. 工单可以创建、完成和刷新保留。
7. 决策报告可以正常显示。
8. AI 助手如果要演示，提前确认模型可用。
9. 浏览器没有打开隐私内容。
10. 麦克风声音清楚，录屏画面完整。

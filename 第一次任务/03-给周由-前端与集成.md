# 给周由：第一次任务说明

你这轮负责的是**独立前端展示包**。

你的目标不是碰后端或改数据规则，而是把 `frontend/` 这一块做成一个结构清楚、页面更顺、交互更完整、可以单独交回给许奕整合的前端成果。

## 你的写入边界

你这轮**只允许**改这些：

- `frontend/`

你这轮**不要碰**这些：

- `backend/`
- `knowledge_base/`
- `data/`
- `docs/`
- `scripts/`

如果你觉得某个接口返回不合理：

不要自己改后端，直接把问题写在说明里交给许奕。

## 你这轮的独立任务

### 任务 A：把当前前端整理成真正的展示工作台

你现在不是从零开始，所以第一轮最合理的事情是把现有页面继续整理清楚。

建议你重点做：

- 总览页更清楚
- 数据浏览页更清楚
- 分析页更清楚
- 问答区更清楚

重点不是页面数量增加，而是页面结构更像可演示产品。

### 任务 B：把大页面进一步拆成小组件

这一步很适合独立开发，因为你完全可以只在 `frontend/` 内完成。

建议新增的组件方向：

- `frontend/src/components/AppHeader.vue`
- `frontend/src/components/TabNavigation.vue`
- `frontend/src/components/FilterToolbar.vue`
- `frontend/src/components/AnalyticsSummary.vue`
- `frontend/src/components/StatusBanner.vue`

你不一定非要用这些名字，但建议朝“拆组件、减主页面负担”这个方向走。

### 任务 C：把页面状态补完整

你这轮应该补这些前端细节：

- 加载中状态
- 无数据状态
- 请求失败状态
- 接口未启动时的提示

这些东西看起来小，但对于老师演示印象很重要。

### 任务 D：如果你有余力，再做一个最小可用图表

如果你觉得第一轮已经适合接图表，可以只做一个最小版本。

要求：

- 只在 `frontend/` 内完成
- 不改 API 契约
- 不为了接图表把整个前端结构弄乱

如果不接图表也没关系，优先把结构和联调做顺。

建议至少新增 2 到 4 个组件文件，例如：

- `frontend/src/components/AppHeader.vue`
- `frontend/src/components/TabNavigation.vue`
- `frontend/src/components/FilterToolbar.vue`
- `frontend/src/components/StatusBanner.vue`

## 你交回来的成果必须满足什么

你交回来时，我希望它已经像一个**独立前端展示包**。

也就是说，许奕拿到后应该能直接知道：

1. 你主要优化了哪些页面区域
2. 哪些组件是新拆出来的
3. 哪些地方已经接了真实接口
4. 哪些地方还是占位，但结构已经定好了

## 你交回来时必须附上的说明

请明确写：

- 你改了哪些文件
- 你新增了哪些组件
- 哪些页面已经能稳定展示
- 哪些地方你故意没动，留给许奕统一整合

## 给你的建议

- 第一轮重点是“整洁、稳定、可展示”，不是“炫”
- 最好只在 `frontend/` 内完成所有工作
- 不要自己适配出一套和后端契约不同的字段名
- 你越像在做一个独立前端包，许奕整合起来就越轻松
- 看一下 [`docs/10-data-source-research.md`](../docs/10-data-source-research.md)，前端文案可以明确表述为“参考公开建筑能耗数据与 benchmark 口径构建”，不要把当前数据说成纯随机生成

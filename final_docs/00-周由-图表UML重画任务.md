## 0. 通用规范（先看这一段）

1. **内容不许改**：节点、文字、流程、字段、数字一律照搬现有 Mermaid 图，只换记号和美化。有疑问先问，不要自行增删业务逻辑。
2. **工具建议**：
   - UML 类图 / 用例图 / 状态机 / 顺序图 / 部署图 / 活动图 → 用 **StarUML 或 draw.io（UML 模板）**，或 **PlantUML**（写文本批量出图最省事）。
   - 经济/管理类图（柱状、折线、甘特、象限、画布）→ 用 **draw.io / Excel / Visio / PPT** 画好后导出。
3. **产出格式**：每张图导出 **PNG（建议宽 ≥ 1600px）**，统一放到 `final_docs/images/`，命名规则 `文档-编号-图名.png`，例如 `srs-fig01-usecase.png`、`sdd-fig06-state-workorder.png`。
4. **回填文档**：在原 Mermaid 代码块上方或下方用图片引用 `![图名](images/xxx.png)`。**先不要删 Mermaid 源码**，保留作为对照，等许奕确认后再决定是否删除。
5. **优先级**：先做 **P0（评审一眼看出"不是 UML"的图）**，再做 P1，最后 P2（仅美化）。时间不够先保 P0。

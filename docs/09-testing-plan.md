# 测试计划

## 1. 测试目标

确保系统在课程演示场景下具备基本可用性和稳定性，优先覆盖查询、分析、问答、MCP 协议入口和页面联调主流程。

## 2. 测试范围

- 后端接口可访问性
- 数据加载与字段完整性
- 主要统计逻辑结果格式
- 前端页面基础渲染
- 问答接口响应结构
- MCP Tools、MCP Resources 和 stdio 客户端集成

## 3. 测试层次

### 3.1 后端接口测试

已在 `backend/tests/` 中预留：

- 健康检查测试
- 数据接口测试
- 分析接口测试
- 问答接口测试
- MCP Server 对象级测试
- MCP stdio 客户端集成测试

### 3.2 人工联调测试

- 打开首页是否能看到总览
- 是否能筛选建筑并看到记录变化
- 是否能看到 COP 排名和异常列表
- 问答输入后是否返回回答与引用
- MCP stdio 模式是否可由客户端初始化、列工具并调用工具
- MCP streamable-http 模式是否能正常启动并监听 `/mcp`

### 3.3 演示彩排测试

- 后端启动后接口是否稳定
- 前端页面是否无明显报错
- 演示脚本是否与页面一致
- 是否存在空白页面、死链接或错误文案
- MCP 启动脚本是否能解释 stdio 模式下终端安静等待的原因

## 4. 当前验证结果

- 后端完整测试：`75 passed, 1 warning`
- MCP 专项测试：`6 passed`
- 前端生产构建：已通过 Vite 构建
- 真实浏览器验收：已覆盖总览、数据浏览、统计分析、工单、决策报告和智能问答

## 5. 当前已知限制

- FastAPI/TestClient 依赖链存在弃用提示，不影响当前运行。
- 前端存在 `favicon.ico 404` 和 ECharts 兼容提示，属于非阻断改进项。
- 30 分钟连续运行稳定性尚未进行严格计时压测。

# 第一次任务包

这个文件夹是给第一次任务分发专门准备的。你可以直接把对应文件发给对应同学，他们只需要按自己那一份做，不要到处看、到处改。

## 这个文件夹里有什么

- [`00-统一说明.md`](00-统一说明.md)：第一轮任务的共同背景、目标和边界
- [`01-给王天一-后端与架构.md`](01-给王天一-后端与架构.md)：王天一的第一次任务
- [`02-给马源胜-数据与AI.md`](02-给马源胜-数据与AI.md)：马源胜的第一次任务
- [`03-给周由-前端与集成.md`](03-给周由-前端与集成.md)：周由的第一次任务
- [`04-统一提交要求.md`](04-统一提交要求.md)：三个人统一遵守的提交和说明要求
- [`05-给许奕-整合说明.md`](05-给许奕-整合说明.md)：许奕自己整合时参考的说明

## 你怎么用

1. 先把 [`00-统一说明.md`](00-统一说明.md) 和 [`04-统一提交要求.md`](04-统一提交要求.md) 发给三个人。
2. 再分别把对应人的任务文件单独发给他。
3. 明确告诉他们：第一轮不是做最终版，而是做“独立成果包”。
4. 第一轮结束后，你按 [`05-给许奕-整合说明.md`](05-给许奕-整合说明.md) 去收成果。

## 这轮任务的核心原则

- 不追求一次做完全部功能。
- 不允许随便改字段名、接口路径和主页面结构。
- 每个人只改自己负责的目录。
- 如果要改公共规则，先找你确认，再动手。

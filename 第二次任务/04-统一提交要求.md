# 第二次任务统一提交要求

这轮三个人交回来的东西，必须继续满足“**可独立验收、可独立说明、可由许奕统一接回主仓库**”这三个条件。

## 1. 不要交零散改动

不要交那种：

- 我改了很多地方，你自己看
- 我顺手改了几个公共文件，应该没关系
- 我觉得那个文档也该一起改，就一起改了

第二轮还是和第一轮一样：

**每个人只交自己负责目录里的独立成果包。**

## 2. 每个人都必须附交付说明

建议：

- 王天一：`backend/BACKEND_ROUND2_DELIVERY.md`
- 马源胜：`data/processed/DATA_AI_ROUND2_DELIVERY.md`
- 周由：`frontend/FRONTEND_ROUND2_DELIVERY.md`

说明里至少写清楚：

1. 你改了哪些文件
2. 你新增了哪些文件
3. 你这轮最核心的增强是什么
4. 哪些地方你故意没动
5. 许奕整合时应该先看哪里

## 3. 第二轮特别强调的边界规则

- 王天一可以新增后端能力，但不要改坏旧接口
- 周由第二轮默认只使用现有稳定接口，不依赖王天一新增接口
- 马源胜继续不要改主样例数据和数据字典

## 4. 交回前自己至少检查一次

### 王天一

- 后端测试有没有补上
- 旧接口有没有回归
- 新能力是不是只在 `backend/` 内完成

### 马源胜

- 新文档是不是可直接引用
- 结构化资料是不是足够清楚
- 有没有越界去碰主数据和公共文档

### 周由

- 前端 build 能不能过
- 说明里写的东西是不是代码里真的有
- 有没有偷偷在前端适配出一套和后端不一致的字段名

## 5. 许奕最后最看重什么

第二轮许奕最看重的，不是你“写了多少”，而是：

1. 你是不是守住了边界
2. 你的成果是不是独立完整
3. 你的成果能不能直接并进现有统一仓库
4. 你的增强是不是对第二次整合真的有帮助

---

## 1. 概述

本文档提供建筑能耗异常的诊断思路和排查步骤，主要面向以下场景：

- 系统自动标记 `equipment_status=abnormal` 的记录
- COP 低于预设阈值的时段
- 电耗超过建筑历史均值 1.5 倍标准差的时段
- 深夜/低人员密度时段出现异常高能耗的情况

---

## 2. 常见异常类型与原因

### 类型 A：非工作时段高能耗异常

**特征：**
- 时段：00:00-06:00 或 21:00 后
- occupancy_density_per_100m2 < 10
- electricity_kwh 高于该建筑深夜均值 30% 以上

**常见原因（参考 SHIFDR 设备行为观察）：**

| 原因 | 概率 | 说明 |
|---|---|---|
| 设备未关机或误触发 | 高 | 空调/新风机在无人时段继续运行，BAS 调度逻辑故障 |
| 网络设备/服务器持续运行 | 中 | 实验楼/图书馆的 IT 设备负载持续 |
| 照明控制失效 | 中 | 楼道/机房照明未自动关闭 |
| 数据采集异常 | 低 | 传感器漂移导致读数偏高（需交叉验证其他字段） |

**排查步骤：**
1. 确认 `occupancy_density_per_100m2` 是否确实低于 10
2. 查看同时段 `hvac_kwh` 占 `electricity_kwh` 的比例；若比例异常高，优先排查空调系统
3. 确认 `chilled_water_supply_temp_c` 是否在正常范围（6～8°C）
4. 检查是否存在 demand response 事件（当前数据集暂无此标记）
5. 记录并上报：时间段、建筑、电耗数值、现场确认结果

**典型案例（来自当前样例数据）：**
- R00001：综合教学楼A，2026-03-01 00:00，electricity=282.29 kWh，occupancy=4.1，equipment_status=abnormal
- R00642：综合教学楼A，2026-03-21 03:00，electricity=286.0 kWh，凌晨异常高值

---

### 类型 B：制冷效率低下（低 COP）异常

**特征：**
- COP = cooling_load_kwh / hvac_kwh < 2.5
- 设备状态可能标记为 abnormal，也可能仍为 normal

**常见原因：**

| 原因 | 说明 |
|---|---|
| 冷水机组结垢 | 换热效率下降，导致相同制冷量需要消耗更多电能 |
| 制冷剂泄漏或不足 | 制冷循环效率直接下降 |
| 冷却水温度偏高 | 夏季工况下冷却水散热不足，影响冷凝效果 |
| 部分负荷低效运行 | 机组在远离额定工况下运行，效率显著下降 |
| 温差 ΔT 不合理 | 供回水温差过小（< 3°C）表明负荷匹配或流量设置问题 |

**排查步骤：**
1. 计算当前时段 COP，确认低于 2.5 阈值
2. 检查 `chilled_water_supply_temp_c` 是否高于 9°C（偏高）
3. 检查温差 ΔT = `return_temp - supply_temp`；若 < 3°C，优先排查水量调节
4. 对比同一建筑历史 COP 均值，确认是趋势性下降还是单点异常
5. 安排设备巡检：冷水机组冷凝器清洗、制冷剂检漏

**典型案例（来自当前样例数据）：**
- R00024：图书信息楼C，2026-03-01 21:00，COP ≈ 2.34，当前数据集最低值，equipment_status=abnormal

---

### 类型 C：超高电耗异常（峰值超过历史最大值 20%）

**特征：**
- `electricity_kwh` 超过该建筑历史最大值
- 可能发生在工作时段（09:00-18:00），人员密度高

**常见原因：**

| 原因 | 说明 |
|---|---|
| 大规模实验设备同时开机 | 实验楼特有，如高功率仪器集中使用 |
| 极端天气导致空调超负荷 | 气温突破历史高点时 HVAC 持续满负荷运转 |
| 临时活动（大规模会议/展览） | 人员密度突增，照明+空调同时满载 |
| 电气设备短路或漏电 | 安全问题，优先排查 |

**排查步骤：**
1. 确认 `environment_temp_c` 是否处于极端值
2. 确认 `occupancy_density` 是否处于极高值
3. 查看 `hvac_kwh / electricity_kwh` 比例：若比例正常，考虑非空调设备原因
4. 紧急情况下：联系楼管人员现场确认用电设备状态

**典型案例（来自当前样例数据）：**
- R00606：科研实验楼D，2026-03-19 15:00，electricity=561.85 kWh（全库最高）

---

### 类型 D：数据采集异常（传感器故障/漂移）

**特征（当前样例数据暂无此类型，但在实际场景中常见）：**
- `electricity_kwh` 与 `hvac_kwh` 比例严重失常（如 hvac_kwh > electricity_kwh）
- 温度字段出现明显不合理值（如 supply_temp > return_temp）
- 某字段突然变为 0 或负值

**排查步骤：**
1. 交叉验证多个字段的物理关系（hvac_kwh ≤ electricity_kwh）
2. 对比前后时间段的数值连续性
3. 标记为"数据质量问题"，不纳入异常统计
4. 通知数据采集团队检查传感器

## 3. 异常判定阈值汇总（演示用规则）

| 指标 | 正常范围 | 告警阈值 | 告警级别 |
|---|---|---|---|
| COP | ≥ 3.0 | < 2.5 | 中 |
| electricity_kwh（3h） | ≤ 建筑历史均值 × 1.5 | > 建筑历史均值 × 1.5 | 中 |
| 深夜电耗（00-06:00） | ≤ 建筑日均的 60% | > 建筑日均的 80% | 低 |
| 供水温度（supply_temp） | 6 ～ 8°C | > 9°C 或 < 5°C | 高 |
| 温差 ΔT | 4 ～ 6°C | < 3°C 或 > 7°C | 中 |
| hvac_kwh / electricity_kwh 比 | 30% ～ 40% | > 50% 或 < 20% | 低 |

**注：** 以上阈值为演示级规则，基于当前样例数据统计得出，不代表工程实际验收标准。
